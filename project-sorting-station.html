Sorting Station — Vision-Based Classification System

Vision-based sorting system that classifies plastic parts by type in real time, built from scratch using TIA Portal V17, PLCSIM (S7-1500), and Factory I/O. Reactive classification logic, edge detection, and conveyor flow control — all in SCL.

The Problem

In manufacturing lines that produce multiple plastic part types — bases, lids, bodies — parts need to be sorted automatically for downstream assembly or packaging. Manual sorting is slow, error-prone, and doesn't scale. The challenge in PLC logic is handling detection timing, conveyor coordination, and classification without mixing up parts or causing collisions.

The Solution

A complete PLC program that reads a vision sensor, identifies the part type, activates the correct sorting diverter, and manages conveyor flow to prevent part collision — all while tracking production counts per part type in real time.

How It Works

1
Entry conveyor feeds parts toward the vision sensor
2
Vision sensor reads the part and returns an integer ID — 1 = base, 2 = lid, 3 = body
3
Edge detection captures the part type on the rising edge (0 → value) — one detection per part
4
Entry conveyor stops once the detected part clears the sensor — prevents the next part from arriving during sorting
5
Sorter activates — the corresponding diverter turns to route the part to its designated ramp
6
At_Exit sensor confirms the part entered the ramp (rising edge detection)
7
Sorter deactivates, entry conveyor restarts — system ready for the next part

Program Structure

FC1 — Control
Ladder (LAD)

FC2 — Classification
SCL
Core sorting logic — DINT-to-INT sensor conversion, edge detection, conveyor flow control, diverter activation, exit confirmation, per-type counting.
FC3 — Monitoring
SCL
Production analytics — machine state tracking, parts per minute, distribution percentages by part type.

Key Technical Decisions

  • DINT vision sensor — Factory I/O outputs a 32-bit DINT requiring explicit DINT_TO_INT conversion. Implicit conversion caused runtime warnings and unpredictable behavior
  • Entry conveyor as flow control — instead of a mechanical stop blade, the entry conveyor stops after the part clears the sensor. Key logic: Entry_Conveyor := NOT SortingActive OR visionActive
  • Edge detection on At_Exit — the exit sensor stays TRUE while a part is on it. Without edge detection, SortingActive resets every scan. AtExitPrev flag ensures single-trigger
  • Separate safety layer — FC1 handles all safety independently from FC2. Emergency stop forces all outputs OFF regardless of classification state

Lessons Learned

Type mismatches crash PLCs
A DINT sensor value written to an INT variable without conversion caused repeated PLC faults. Always match types explicitly in Siemens SCL — implicit conversion is not safe.
Array bounds matter
An array declared as [0..1] instead of [0..9] caused runtime crashes when accessing index 2+. PLCSIM Standard stops on out-of-range access with no descriptive error.
Conveyor flow beats delay timers
The gap between detection and physical arrival at the sorter is the hardest problem. Controlling conveyor flow (stop/start) is more reliable and scalable than trying to engineer precise time delays.
Sensors can be always-active
Assuming a sensor is normally FALSE is dangerous. At_Exit was TRUE by default in Factory I/O, causing immediate classification reset on startup. Edge detection solved it completely.
